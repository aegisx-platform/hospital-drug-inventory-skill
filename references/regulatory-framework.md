## 1. โครงสร้างกฎหมาย 5 ชั้น {#legal-layers}

```
ชั้น 1: พ.ร.บ.จัดซื้อจัดจ้างและการบริหารพัสดุภาครัฐ พ.ศ. 2560
        + กฎกระทรวง 2563 (พัสดุที่รัฐส่งเสริม/บัญชีนวัตกรรม)
        + ระเบียบกระทรวงการคลัง 2560

ชั้น 2: ระเบียบ สธ. ว่าด้วยการบริหารจัดการด้านยาและเวชภัณฑ์ที่มิใช่ยา
        ของส่วนราชการและหน่วยงานในสังกัด พ.ศ. 2563
        (ปรับปรุงจากฉบับ 2557)

ชั้น 3: ประกาศ สธ. เรื่อง เกณฑ์จริยธรรมการจัดซื้อจัดหาและ
        การส่งเสริมการขายยาและเวชภัณฑ์ที่มิใช่ยา พ.ศ. 2564
        (ยกเลิกฉบับ 2557)

ชั้น 4: นโยบายพัฒนาประสิทธิภาพระบบบริหารเวชภัณฑ์ (ตั้งแต่ 2542)
        + หลักเกณฑ์ฯ กองบริหารการสาธารณสุข 2563

ชั้น 5: ระบบราคาอ้างอิง DMSIC (อัพเดททุก 3-4 เดือน)

ชั้น 6: ประกาศ สธ. เรื่อง มาตรฐานข้อมูลบริหารเวชภัณฑ์ (ยา) พ.ศ. 2568
        → ส่งข้อมูล 5 แฟ้มทุกเดือนผ่าน API ไปยัง DMSIC
        → ดูรายละเอียดที่ `dmsic-5-file.md`
```

### กฎหมายแม่บท — พ.ร.บ.จัดซื้อจัดจ้าง 2560

3 วิธีจัดซื้อหลัก:
- **วิธีประกาศเชิญชวนทั่วไป** (e-bidding) — วงเงินสูง, เปิดกว้าง
- **วิธีคัดเลือก** — เฉพาะกรณีตามกฎกระทรวง
- **วิธีเฉพาะเจาะจง** — วงเงินไม่เกิน 500,000 บาท หรือกรณีพิเศษ

### กฎกระทรวง 2563 — พัสดุที่รัฐส่งเสริม

- บัญชียาหลักแห่งชาติ — จัดซื้อตามระเบียบ ว 439
- บัญชีนวัตกรรมไทย — สิทธิพิเศษจัดซื้อโดยวิธีเฉพาะเจาะจง
- วัคซีน — ระเบียบเฉพาะตามคณะกรรมการวัคซีนแห่งชาติ 2563

## 2. กระบวนการหลัก 5 ขั้นตอน {#five-processes}

ตามกรอบมาตรฐานกองบริหารการสาธารณสุข:

```
1. คัดเลือกรายการยา (Selection)
   └─ PTC พิจารณา → กรอบบัญชียาร่วมจังหวัด → ED/NED/VEN classification

2. ประมาณการ (Estimation)
   └─ ใช้ข้อมูลย้อนหลัง 3 ปี → วิเคราะห์แนวโน้ม → คำนวณปริมาณจัดซื้อ
   └─ Min stock = ปริมาณใช้ต่อปี ÷ 12
   └─ Max stock = Min × 1.5
   └─ คงคลังไม่เกิน 2 เดือน

3. จัดซื้อ (Procurement)
   └─ จัดทำแผนจัดซื้อประจำปี → เสนอ คกก. → สสจ. → e-GP
   └─ PR → เจรจาราคา (อ้างอิง DMSIC) → PO → ตรวจรับ (GR)
   └─ ต้องจัดซื้อด้วยราคาสุทธิเท่านั้น

4. เก็บและสำรอง (Storage)
   └─ รพ. เป็นคลังสำรองระดับอำเภอ
   └─ ระบบจัดเก็บ + ตรวจสอบคุณภาพ + ปลอดภัย
   └─ FEFO + ตรวจ stock สัปดาห์ละครั้ง

5. กระจาย (Distribution)
   └─ คลังใหญ่ → คลังย่อย (ward/OPD/ER) → ผู้ป่วย
   └─ Utilization Evaluation ตามข้อ 13 ของระเบียบ 2563
```

## 3. โครงสร้างคณะกรรมการ {#committee-structure}

### ระดับ รพ. (ข้อ 6 ระเบียบ 2563)

```
คณะกรรมการเภสัชกรรมและการบำบัด (PTC)
├── คัดเลือกรายการยาเข้า formulary
├── กำหนดนโยบายการใช้ยา
└── Utilization Evaluation (ข้อ 13)

คณะกรรมการบริหารเวชภัณฑ์ (คกก.ข้อ 6)
├── กำหนดบัญชีรายการยาและเวชภัณฑ์
├── ควบคุมกำกับ การจัดซื้อตามแผน
├── รับผิดชอบ: กำหนดความต้องการ, จัดซื้อ, ควบคุมเก็บรักษา, เบิกจ่าย,
│   ตลอดจนการชี้แจงและวิเคราะห์การใช้เวชภัณฑ์
└── อนุมัติแผนปฏิบัติการจัดซื้อประจำปี
```

### ระดับจังหวัด/เขต

```
สำนักงานสาธารณสุขจังหวัด (สสจ.)
├── จัดทำกรอบบัญชียาร่วมจังหวัด (ข้อ 7)
├── รับแผนจัดซื้อจาก รพ. พิจารณาอนุมัติ
└── จัดซื้อร่วมระดับจังหวัด (ข้อ 9)

คณะกรรมการร่วมระดับจังหวัด/เขต
├── จัดซื้อร่วม (pooled procurement)
├── ต่อรองราคากลาง
└── สอดคล้องกับบัญชียาร่วมจังหวัด
```

### ระดับชาติ

```
ศูนย์ข้อมูลข่าวสารด้านเวชภัณฑ์ (DMSIC)
├── จัดทำราคาอ้างอิง ทุก 3-4 เดือน
├── รวบรวม/วิเคราะห์ความก้าวหน้าตามมาตรการ
└── เสนอกระทรวง

สำนักตรวจราชการ กระทรวงสาธารณสุข
├── ควบคุมกำกับติดตาม นิเทศ
└── ตรวจราชการตามตัวชี้วัด

คณะกรรมการพัฒนาระบบยาแห่งชาติ
├── เกณฑ์จริยธรรมส่งเสริมการขายยาระดับชาติ
└── กลไกต่อรองราคายา
```

## 4. ระเบียบ สธ. 2563 — สาระสำคัญ {#regulation-2563}

### ข้อกำหนดที่ระบบต้องรองรับ

| ข้อ | สาระสำคัญ | ผลต่อระบบ |
|-----|-----------|-----------|
| 6 | บริหารโดย คกก.ร่วม ระดับอำเภอ/จังหวัด/เขต | ต้องมี committee approval workflow |
| 7 | สสจ. จัดทำกรอบบัญชียาร่วมจังหวัด | Drug formulary ต้องรองรับ provincial shared list |
| 8 | จัดทำแผนปฏิบัติการจัดซื้อประจำปี เสนอ คกก. + สสจ. | Annual procurement plan module |
| 9 | จัดซื้อร่วมระดับจังหวัด/เขต ได้ | Pooled procurement support |
| 10 | ยาเวชภัณฑ์ที่จัดซื้อด้วยราคาสุทธิ + สัญญาราคาคงที่ | **ราคาสุทธิเท่านั้น ไม่มีส่วนลด/rebate** |
| 11 | **ผู้จัดซื้อ กับ ผู้ควบคุมคลัง ต้องเป็นคนละคน** | Separation of duties enforcement (RBAC) |
| 12 | ส่วนราชการ รายงาน สสจ./ปลัด สธ. | Reporting module ตามลำดับขั้น |
| 13 | Utilization Evaluation — ประเมินประสิทธิผล คุณค่า ความปลอดภัย | Drug utilization review module |
| 14 | เขต/ส่วนราชการ สามารถกำหนดแนวทางเพิ่มเติม | Configurable rules per region |

### ข้อ 10 — ราคาสุทธิ (สำคัญมาก)

```
ยาและเวชภัณฑ์ที่มิใช่ยาแต่ละรายการให้ดำเนินการจัดซื้อตามแผนปฏิบัติ
การจัดซื้อยาและเวชภัณฑ์ที่มิใช่ยาประจำปี
ในวงเงินที่กำหนด และสัญญาจะซื้อจะขายแบบราคาคงที่เท่านั้น
ยกเว้นกรณีจำเป็นเร่งด่วน
```

**ผลต่อ data model:** PO ต้องเก็บราคาสุทธิ (`netPrice`) ไม่มี discount/rebate fields

### ข้อ 11 — Separation of Duties

```
ควบคุมตรวจสอบซึ่งกันและกัน ระหว่างเจ้าหน้าที่จัดซื้อ
และผู้ที่ทำหน้าที่ควบคุมคลังยาและเวชภัณฑ์ที่มิใช่ยา
ให้จัดกลุ่ม สำหรับหน่วยงานที่มีความพร้อมด้านบุคลากร
ให้แยกหน่วยจัดซื้อและหน่วยคลังยาและเวชภัณฑ์ที่มิใช่ยา
ออกจากกันได้ เพื่อให้มีระบบความรับผิดชอบและตรวจสอบ
ที่แยกออกจากกันชัดเจน
```

**ผลต่อ RBAC:**
```typescript
// ห้ามมี role เดียวกันทั้ง PURCHASER + WAREHOUSE_KEEPER
enum DrugRole {
  PURCHASER,          // ผู้จัดซื้อ
  WAREHOUSE_KEEPER,   // ผู้ควบคุมคลัง
  INSPECTOR,          // ผู้ตรวจรับ (กรรมการตรวจรับ)
  APPROVER,           // ผู้อนุมัติ (หัวหน้า/ผอ.)
  PHARMACIST,         // เภสัชกร (จ่ายยา)
}

// Business rule: user CANNOT have both PURCHASER + WAREHOUSE_KEEPER
```

## 5. เกณฑ์จริยธรรม 2564 — ข้อกำหนดสำคัญ {#ethics-2564}

ประกาศกระทรวงสาธารณสุข ราชกิจจานุเบกษา 15 พ.ค. 2564
(ยกเลิกฉบับ 2557)

### ข้อกำหนดที่กระทบระบบจัดซื้อ

| หมวด | ข้อ | ข้อกำหนด | ผลต่อระบบ |
|------|-----|----------|-----------|
| 1 (ผู้สั่งใช้) | 13 | **สั่งยาด้วยชื่อสามัญทางยา** | Generic name เป็น primary ใน drug order |
| 2 (ผู้บริหาร) | 15(2) | **จัดซื้อด้วยราคาสุทธิ** | ไม่มี discount/rebate ในระบบ |
| 2 | 15(3) | **ห้ามผลประโยชน์ต่างตอบแทนทุกประเภท** | ไม่มี commission tracking |
| 2 | 15(5.4) | **ระบบบริหารความเสี่ยงทุจริต + ตรวจสอบภายใน** | Audit trail ครบทุก transaction |
| 4 (รพ.) | 20(2) | เกณฑ์จริยธรรมผู้จำหน่ายประกอบการคัดเลือก | Vendor ethics rating optional |
| 4 | 20(6) | รับสนับสนุนจัดประชุมวิชาการ ต้องเปิดเผยทุกครั้ง | Sponsorship disclosure log |

### ข้อ 13 — สั่งยาด้วยชื่อสามัญ

ส่งผลต่อ Drug Master Data:
- `genericName` (ชื่อสามัญ) เป็น primary display/search
- `tradeName` (ชื่อการค้า) เป็น secondary
- Drug Order screen ค้นหาจาก generic name ก่อน

### หลัก 6 ด้าน ธรรมาภิบาลระบบยา

1. ความโปร่งใส (transparency)
2. ความรับผิดรับชอบ (accountability)
3. นิติธรรม (rule of law)
4. คุณธรรม (ethics)
5. คุ้มค่า (value for money)
6. การมีส่วนร่วม (participation)

## 6. ราคาอ้างอิง DMSIC {#dmsic-reference-price}

ศูนย์ข้อมูลข่าวสารด้านเวชภัณฑ์ กระทรวงสาธารณสุข (DMSIC)

- จัดทำบัญชีราคาอ้างอิงยาและเวชภัณฑ์
- แจ้งหน่วยงานทุก **3-4 เดือน**
- ใช้เป็นข้อมูลประกอบการพิจารณาจัดซื้อ + กำกับราคา
- URL: https://dmsic.moph.go.th

**ผลต่อระบบ:**
```typescript
// Drug pricing must track reference price
interface DrugPricing {
  drugId: string;
  dmiscReferencePrice: number;  // ราคาอ้างอิง DMSIC
  contractPrice: number;        // ราคาตามสัญญา
  lastPurchasePrice: number;    // ราคาจัดซื้อล่าสุด
  medianPrice3Years: number;    // ค่ากลาง 3 ปี
  priceEffectiveDate: Date;
}

// Alert: จัดซื้อราคาสูงกว่าราคาอ้างอิง DMSIC
if (purchasePrice > dmiscReferencePrice * 1.0) {
  // flag for review — ต้องมีเหตุผลประกอบ
}
```

## 7. ตัวชี้วัด 16 ตัว {#kpis}

จากงานวิจัย กองบริหารการสาธารณสุข (2567)
แบ่งตาม 5 กระบวนการ:

### คัดเลือก (2 ตัว)
- ร้อยละรายการยาในบัญชียาหลักแห่งชาติ (ED ratio)
- ร้อยละมูลค่ายาในบัญชียาหลักแห่งชาติ

### ประมาณการ (4 ตัว)
- ความแม่นยำของการประมาณการ (forecast accuracy)
- อัตราคงคลัง (stock months) — ไม่เกิน 2 เดือน
- อัตรายาหมดอายุ/เสื่อมสภาพ
- อัตรายาขาดคลัง (stockout rate)

### จัดซื้อ (3 ตัว)
- ร้อยละการจัดซื้อตามแผน
- ร้อยละการจัดซื้อร่วม (pooled procurement ratio)
- ราคาจัดซื้อเทียบราคาอ้างอิง DMSIC

### เก็บและสำรอง (6 ตัว)
- อัตราความถูกต้องของคลัง (inventory accuracy)
- อัตราการหมุนเวียนสินค้า (turnover rate)
- มูลค่ายาคงคลัง (inventory value)
- อัตรายาใกล้หมดอายุ (< 6 เดือน)
- อัตราสูญเสียจากยาหมดอายุ
- ร้อยละรายการที่มี min stock ต่ำกว่าเกณฑ์

### กระจาย (1 ตัว)
- ร้อยละรายการยาที่ขาดขณะให้บริการผู้ป่วย

### มิติวัดผล
- คุณภาพ: 10 ตัว
- เวลา: 2 ตัว
- การเงิน/ต้นทุน: 9 ตัว
- ผลิตภาพ: 6 ตัว
- ตอบนโยบาย: 8 ตัว

## 8. Timeline ปฏิทินงบประมาณ {#annual-timeline}

### ปฏิทินการจัดทำแผนจัดซื้อ (ตามเอกสารแนบท้ายระเบียบ 2563)

```
ก.ค. - ส.ค.    ── จัดทำข้อมูลฐาน ──────────────────
                ├── รวบรวมข้อมูลใช้ยา 3 ปีย้อนหลัง
                ├── วิเคราะห์แนวโน้มเปลี่ยนแปลง
                ├── ประมาณการใช้ในปีงบใหม่
                ├── สำรวจยาคงคลัง (1-15 ก.ค.)
                ├── กำหนดปริมาณขั้นต่ำสำรอง
                ├── รวบรวมราคาอ้างอิง/ราคากลาง
                └── กำหนดวงเงินจัดซื้อ

ส.ค. - ก.ย.    ── เสนอแผนและพิจารณา ─────────────
                ├── จัดทำแผนปฏิบัติการจัดซื้อ
                ├── เสนอ คกก.บริหารเวชภัณฑ์ (ข้อ 6)
                ├── หัวหน้าหน่วยงาน/ผอ. เห็นชอบ
                ├── เสนอ สสจ. (ส่วนภูมิภาค)
                └── ประกาศแผนใน e-GP

ต.ค.           ── เริ่มปีงบใหม่ ──────────────────
                └── ดำเนินการจัดซื้อตามแผน

ทุกไตรมาส      ── ควบคุมกำกับ ──────────────────
                ├── Q1: ต.ค. - ธ.ค.
                ├── Q2: ม.ค. - มี.ค.
                ├── Q3: เม.ย. - มิ.ย.
                └── Q4: ก.ค. - ก.ย.
                สรุปผลและรายงานรายไตรมาส

ต.ค. (ปีถัดไป) ── ประเมินผลการดำเนินการ ───────────
```

### การปรับแผนระหว่างปี (ข้อ 3 เอกสารแนบท้าย)

กรณีต้องขออนุมัติปรับแผน:
1. **รายการยาใหม่ที่ไม่ได้กำหนดอยู่ในแผน** — ต้องเสนอเพิ่มรายการ + วงเงิน
2. **อัตราใช้เพิ่มสูงกว่าที่ว่าไว้** หรือเงินรวมในการจัดซื้อตามแผนไม่เพียงพอ — ต้องขออนุมัติเพิ่มเงิน

ขั้นตอน:
```
หน่วยงานเสนอ → สสจ./หัวหน้าส่วนราชการ กำหนดช่วงระยะเวลา →
หน่วยงานพิจารณาอนุมัติปรับแผนได้ → แจ้ง สสจ.
```

## 10. Implications สำหรับระบบ AegisX {#system-implications}

### Must-Have Features ตามระเบียบ

1. **Annual Procurement Plan module** — จัดทำแผนตามแบบฟอร์ม + timeline + approval workflow
2. **Net Price enforcement** — ราคาสุทธิเท่านั้น ไม่มี discount/rebate
3. **Separation of Duties (RBAC)** — ผู้จัดซื้อ ≠ ผู้ควบคุมคลัง
4. **3-Year Historical Data** — ดึงข้อมูลใช้ยา 3 ปีย้อนหลังสำหรับประมาณการ
5. **DMSIC Reference Price tracking** — เก็บราคาอ้างอิง + alert เมื่อซื้อแพงกว่า
6. **Quarterly Reporting** — สรุป แผน vs จริง รายไตรมาส แยก ED/NED
7. **Plan Revision workflow** — ขอปรับแผนระหว่างปี + approval
8. **Generic Name as primary** — ค้นหา/สั่งยาด้วยชื่อสามัญ
9. **Audit Trail ครบ** — ทุก transaction ต้อง traceable (เกณฑ์จริยธรรม)
10. **Stock Month calculation** — คำนวณ + alert เมื่อ > 2 เดือน
11. **Min/Max stock** — Min = ใช้/ปี ÷ 12, Max = Min × 1.5
12. **Provincial Shared Formulary** — กรอบบัญชียาร่วมจังหวัด
13. **Pooled Procurement support** — จัดซื้อร่วมระดับจังหวัด/เขต
14. **e-GP integration** — ประกาศแผนจัดซื้อ
15. **Utilization Evaluation** — ระบบประเมินการใช้ยา
16. **KPI Dashboard** — 16 ตัวชี้วัดตามมาตรฐาน
17. **DMSIC 5-File Export** — ส่งข้อมูล 5 แฟ้ม (DRUGLIST, PURCHASEPLAN, RECEIPT, DISTRIBUTION, INVENTORY) ทุกเดือนผ่าน API ตามประกาศ สธ. 2568 (ดู `dmsic-5-file.md`)
